## 一、前言
1. 什么是GitHub？
- Github是一个面向开源及私有软件项目的托管平台，因为只支持Git作为唯一的版本库  格式，故名Github。Github提供的功能包括：
- 核心功能
‌代码托管‌：提供存储库（Repository）来托管代码，支持 Git 版本控制系统，开发者可以追踪代码变更、管理分支和合并代码。‌‌
- ‌协作工具‌：通过 Pull Request 实现代码审查，Issues 用于任务跟踪，Wiki 和 Projects 辅助项目管理，支持全球开发者协作。‌‌
- ‌自动化与扩展‌：内置 GitHub Actions 实现持续集成/持续部署（CI/CD），并可通过 Marketplace 集成第三方工具。‌‌
- 与 Git 的关系
Git 是本地运行的分布式版本控制系统，而 GitHub 是基于 Git 的在线平台，提供 Web 界面和社交化协作功能，简化了代码共享与团队协作流程。‌‌
- 应用场景
‌开源项目‌：全球最大的开源社区，开发者可 Fork 项目、提交贡献并参与协作。‌‌
- ‌企业开发‌：支持私有仓库和权限管理，适用于团队开发与代码安全控制

## 二、实际操作
### 🎡<一>、GitHub网站操作（浏览器端-您的文件极少且不包含文件夹时）
1. 登录与准备
    - 访问[Github](https://github.com/)
    - 注册/登录账号（免费账户即可）
    - 右上角点击"+" → "New repository"创建新仓库
2. 创建博客仓库
    - 仓库名格式：“<username>.github.io ”(固定格式，如 johnsmith.github.io)
    - 设为 “Public”（私有仓库Pages功能受限）
    - 勾选 "Add a README file"(可选)
    - 点击 “Create repository”
3. 上传博客文件
   - 进入新建的仓库 → 点击 Add file → Upload files或
   - 拖拽本地博客文件夹（包含HTML/CSS/JS文件）
   - 目录结构要求：<br>
   ├── index.html  # 必须存在<br>
   ├── styles.css<br>
   └── images/
   - 点击 “Commit changes” 提交
4. 启用GitHub Pages
   - 进入仓库 → Settings → Pages
   - Branch：选择 main 或 master
   - Folder：选择 /root（根目录）
   - 保存后生成短链：`https://<username>.github.io`
   - 访问您的博客：`https://<username>.github.io`  
  

**注：1. 此方法只适用于静态网页，如html文件，css文件，js文件等。2.若域名未生效，请检查DNS设置，或稍后再试**  


### ✨<二>、GitHub Desktop操作（图形界面-您的文件较多且包含文件夹时）
1. 安装与登录
   - 访问[Github Desktop](https://desktop.github.com/)
   - 点击`“Download for Windows”`下载安装包
   - 运行安装包，选择安装路径，点击`“Next”`
   - 点击`“Install”`完成安装
   - 登录GitHub账号（File → Options → Accounts）
2. 创建本地博客项目
   - 本地创建博客文件夹（包含HTML/CSS文件）
    my-blog/
     ├── index.html
     └── styles.css
3. 选择本地仓库并提交
   - 点击 File → Add Local Repository
   - 选择博客文件夹 → 点击 Create Repository
   - 填写仓库信息：
       - Name: <username>.github.io  # 固定格式
       - Description: My personal blog
       - 在 Summary 输入提交信息（如 "Initial blog commit"）
       - 点击 Publish repository（首次同步）
       - 勾选 "Keep this code private"（可选）
4. 创建新的仓库并推送
    - 点击 File → New Repository
    - 填写仓库信息：
       - Name: <username>.github.io  # 固定格式
       - Description: My personal blog
       - 在 Summary 输入提交信息（如 "Initial blog commit"）
       - 点击 Publish repository（首次同步）
       - 勾选 "Keep this code private"（可选）
       - 右侧“打开仓库文件夹”，复制或创建您的项目文件（于此）
       - 回到github desktop，右侧发布到仓库
5. 克隆在线仓库
    - 点击`“Clone a repository from the internet”`
    - 输入仓库地址，点击`“Clone”`
    - 选择本地路径，点击`“Clone”`  
    - 对克隆的项目进行操作 
6. 更新博客内容
    - 修改本地文件 → GitHub Desktop自动检测变更
    - 填写提交信息 → 点击 Commit to main
    - 点击 Push origin 同步到GitHub
    - 静态博客时可短链访问：`https://<username>.github.io`

### 🎉<三>、Git命令行操作（终端）
1. 环境配置
```
Bash安装Git（首次使用）
sudo apt install git  # Ubuntu
brew install git     # macOS
```

2. 配置用户信息
```
git config --global user.name "Your Name"
git config --global user.email "email@example.com"
``` 
3. 创建本地博客项目
```
Bash
# 创建项目文件夹
mkdir my-blog && cd my-blog
# 初始化Git仓库
git init
# 创建博客文件
echo "<html><body><h1>My Blog</h1></body></html>" > index.html
touch styles.css
```
4. 连接GitHub仓库
```
Bash
# 在GitHub创建新仓库（名称：<username>.github.io）
# 获取仓库URL（格式：https://github.com/<username>/<username>.github.io.git）
# 添加远程仓库
git remote add origin https://github.com/<username>/<username>.github.io.git
```
5. 提交并推送代码
```
Bash
# 添加所有文件到暂存区
git add .
# 提交变更
git commit -m "Initial blog commit"
# 推送到GitHub（首次使用）
git push -u origin main

# 后续更新推送
git add .
git commit -m "Update blog design"
git push
```

6. 访问博客短链
```
打开浏览器访问：https://<username>.github.io
文件变更后需等待1-2分钟生效
```
